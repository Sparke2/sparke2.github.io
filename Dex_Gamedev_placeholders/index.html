Introduction to the Open4X project

The main goal of Open4X project is to create a 3D multiplayer open-world 4X strategy with continuous gameplay. The game will consist of multiple strategy layers.

Thus, we plan to split the game into its component parts and first implement them separately as demo-games. In fact, the Open4X project will represent three games:

  • Starships Arena - turn-based card game. Tactical layer, combat system reference implementation and balancing tool.
  • Star Cycle - text 2D strategy with a procedurally generated map and event system. Global strategy layer reference implementation.
  • The Main Game - 3D strategy combining elements of the both previous games.

Features

Welcome to a procedurally generated digital universe with its own rules, environments and dangers. Here you are free to pick your own playstyle or change it at any moment. Be a lone traveller, mystery seeker, mercenary or found your own empire. Be vigilant and plan carefully - in this universe, your every action will entail a response from a flexible and fluid ecosystem.

Responsive environment

The key to create immersive universe is to make it responsible. Every action must have its consequences. A player must have the ability and tools to impact processes and ecosystems at will.

Procedural generation

Instead of creating maps, we create laws by which they are created themselves. The universe is then naturally shaped in the process of gameplay.

Creativeness

Players are free to use or not to use any mechanic of the game. Every goal can be achieved in multiple ways. Every tool has multiple purposes.

Interactiveness

Every object must be interactable, changeable and even destructible.

Competitiveness & Cooperation

The basis of the game is the interaction of the players. You can be a lone wolf or found a large interstellar alliance.

Description

At DexGamedev we are primarily engaged in researching game mechanics. Each game is unique in its own way, but it is based on certain mechanics that allow you to create experience for the player. Our goal is to provide players with tools so that they can choose parts of the game they like and filter out those they don't like.

A "game" is, first of all, a set of rules. These rules, mostly mathematical, define the essence of the game, which can be wrapped in a software implementation and a graphical interface.

A game like "chess" is just a set of rules by which pieces move around the board. Chess can have many different implementations - pieces can be large or small, black-white or blue-green, made of plastic, or even digitally displayed on your computer monitor.

The world is developing rapidly and many modern video games exist only thanks to a beautiful picture and excessive marketing. The most successful games live for decades, but when their software part becomes outdated, they will be forgotten. Rarely an old game deserves a re-implementation without any core gameplay modifications.

Our mission is to develop mechanics that can be combined into "immortal" games, which can have many implementations, but the essence of which will be sufficient to make the games interesting for players regardless of the aesthetics or implementation.

Platforms

We use WASM as build target.

Web applications are more and more closely integrated into our lives every year. Thanks to the WASM standard, excellent performance results have been achieved which allows developers to bring features of desktop apps to the web.

We believe that browsers will continue to evolve and the development of browser-executable applications will become a priority, so today we are focusing on developing applications that can be executed as browser applications.
